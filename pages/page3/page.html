2. Семантичні компоненти інтерфейсу

2.1. Шапка (header) з логотипом та навігаційним меню

Реалізовано семантичний тег <header> з наступними елементами:

  • Логотип: <div class="logo"> з зображенням
  • Пошук: <div class="search-bar"> з полем вводу
  • Навігація: <nav class="main-nav"> з <ul>
  • Бургер-меню: <button class="burger-menu-toggle"> для мобільних пристроїв
  • Іконки: профіль, обране та кошик покупок

HTML-код шапки:

<header class="topbar">
    <div class="logo">
        <img src="assets/images/logo.png" alt="Bookerino logo">
    </div>
    <div class="search-bar">
        <svg viewBox="0 0 24 24" aria-hidden="true">...</svg>
        <input type="text" placeholder="Search" aria-label="Пошук">
    </div>
    <div class="top-actions">
        <button class="icon-btn" aria-label="Профіль">...</button>
        <button class="icon-btn" aria-label="Обране">...</button>
        <button class="icon-btn" aria-label="Кошик">...</button>
    </div>
</header>

2.2. Основний блок із розташуванням інформаційних карток

Реалізовано через семантичні теги <section>:

  • Hero-секція: <section id="hero" class="banner"> з заголовком та кнопкою
  • Категорії: <section id="categories"> з картками категорій
  • Товари: <section id="best-seller"> з сіткою карток книг
  • Каталог: <section id="catalog"> з таблицею товарів
  • Відгуки: <section id="feedback"> з формою відгуків

HTML-код основної структури:

<main class="page-container">
    <section id="hero" class="banner">...</section>
    <section id="categories">...</section>
    <section id="best-seller">...</section>
    <section id="catalog">...</section>
    <section id="feedback">...</section>
</main>

2.3. Нижній колонтитул (footer)

Реалізовано семантичний тег <footer> з:

  • Структурованою інформацією у вигляді grid-сітки
  • Розділами: "Контакти", "Інформація", "Каталог"

3. Функціонально-стилістичні вимоги

3.1. Адаптивна навігація з бургер-меню

Реалізація:

HTML-код бургер-меню:

<button class="burger-menu-toggle" aria-label="Меню">
    <span></span>
    <span></span>
    <span></span>
</button>

CSS-стилі:

.burger-menu-toggle {
    display: none;
    flex-direction: column;
    justify-content: space-around;
    width: 2rem;
    height: 2rem;
    background: transparent;
    border: none;
    cursor: pointer;
    z-index: 101;
}

.burger-menu-toggle span {
    width: 100%;
    height: 3px;
    background: var(--primary);
    border-radius: 2px;
    transition: all 0.3s ease;
}

.burger-menu-toggle.active span:nth-child(1) {
    transform: rotate(45deg) translate(0.4rem, 0.4rem);
}

.burger-menu-toggle.active span:nth-child(2) {
    opacity: 0;
    transform: translateX(-1rem);
}

.burger-menu-toggle.active span:nth-child(3) {
    transform: rotate(-45deg) translate(0.4rem, -0.4rem);
}

JavaScript-функціонал: (файл script.js)

(function(){
    const btn = document.querySelector('.burger-menu-toggle');
    const nav = document.querySelector('.main-nav .nav-menu');
    if(!btn || !nav) return;
    
    function toggle(){
        btn.classList.toggle('active');
        nav.classList.toggle('active');
        document.body.classList.toggle('menu-open');
    }
    btn.addEventListener('click', toggle);
})();

Особливості:

  • Бургер-меню з'являється на екранах до 1200px
  • Плавна анімація перетворення іконки при кліку
  • Меню виїжджає зліва з плавним переходом
  • Автоматичне закриття меню при кліку на посилання

3.2. Адаптивна сітка контенту (flexbox/grid)

Використання CSS Grid:

.book-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(min(100%, 11.25rem), 1fr));
    gap: clamp(0.75rem, 2vw, 1rem);
}

Використання Flexbox:

.topbar {
    display: flex;
    align-items: center;
    justify-content: space-between;
    padding: clamp(0.75rem, 2vw, 1rem) clamp(1rem, 4vw, 2rem);
    gap: clamp(1rem, 3vw, 1.5rem);
}

Переваги:

  • Динамічна адаптація кількості колонок залежно від ширини екрана
  • Автоматичне вирівнювання елементів
  • Гнучкість та масштабованість

3.3. Медіа-запити (media queries)

Реалізовані breakpoints:

  • @media (max-width: 1024px): Адаптація для планшетів та менших десктопів
  • @media (max-width: 900px): Адаптація навігації, поява бургер-меню
  • @media (max-width: 768px): Адаптація для мобільних пристроїв
  • @media (max-width: 480px): Адаптація для малих мобільних екранів
  • @media (orientation: landscape): Адаптація для ландшафтної орієнтації

Приклад медіа-запиту:

@media (max-width: 900px) {
    .burger-menu-toggle {
        display: flex;
    }
    
    .main-nav .nav-menu {
        position: fixed;
        top: 0;
        right: -100%;
        width: min(20rem, 80vw);
        height: 100vh;
        flex-direction: column;
        background: var(--card);
        transition: right 0.3s ease;
    }
    
    .main-nav .nav-menu.active {
        right: 0;
    }
}

3.4. Відносні одиниці вимірювання

Використані відносні одиниці:

  • % - для ширини контейнерів та елементів (width: 100%)
  • rem - для розмірів шрифтів та відступів (font-size: 1rem, padding: 1rem)
  • clamp() - для адаптивних розмірів (clamp(0.75rem, 2vw, 1rem))
  • vw/vh - для адаптації до viewport у функції clamp()

Приклади:

.page-container {
    max-width: 1200px;
    margin: 0 auto;
}

.topbar {
    padding: clamp(0.75rem, 2vw, 1rem) clamp(1rem, 4vw, 2rem);
    gap: clamp(1rem, 3vw, 1.5rem);
}

section h2 {
    font-size: clamp(1.25rem, 3vw, 1.5rem);
    margin-bottom: clamp(1rem, 2.5vw, 1.25rem);
}

3.5. Візуальні ефекти та анімації

Реалізовані ефекти:

  • Плавні переходи: transition: transform 0.3s ease, box-shadow 0.3s ease
  • Ефект підняття карток: transform: translateY(-0.5rem) scale(1.02) при hover
  • Зміна тіні: box-shadow: 0 1rem 2.5rem rgba(51, 65, 130, 0.12)
  • Анімація зображень: transform: scale(1.1) при hover
  • Плавна зміна кольорів: transition: color 0.3s ease
  • Анімація бургер-меню: плавне перетворення іконки при кліку з rotate()
  • Keyframe анімації: fadeIn, slideInUp, fadeInLeft, fadeInRight

Приклад анімації:

.book-card {
    transition: transform 0.3s ease, box-shadow 0.3s ease, border-color 0.3s ease;
    opacity: 0;
    animation: fadeInScale 0.5s ease forwards;
}

.book-card:hover {
    transform: translateY(-0.5rem) scale(1.02);
    box-shadow: 0 1rem 2.5rem rgba(51, 65, 130, 0.12);
    border-color: var(--primary);
}

@keyframes fadeInScale {
    from {
        opacity: 0;
        transform: scale(0.9);
    }
    to {
        opacity: 1;
        transform: scale(1);
    }
}

4. Файлова структура веб-застосунку

Організація файлів:

  • HTML-документи: окремо в кореневій директорії
    • index.html - головна сторінка
    • description.html - опис застосунку
    • reports.html - сторінка звітів
  • Стилі: окремий файл styles.css в кореневій директорії
  • Скрипти: окремий файл script.js для функціоналу бургер-меню
  • Графічні ресурси: окрема папка images/ з усіма зображеннями
  • Звіти: окрема папка reports/ з HTML-звітами

Висновок: Файлова структура логічно організована та відповідає вимогам. Всі ресурси зберігаються окремо відповідно до їх призначення.

(function(){

    const nav = document.querySelector('.main-nav .nav-menu');
    if(!btn || !nav) return;
    
    function toggle(){
        btn.classList.toggle('active');
        nav.classList.toggle('active');
        document.body.classList.toggle('menu-open');
    }
    btn.addEventListener('click', toggle);
    
    nav.addEventListener('click', e => {
        if(e.target.closest('a')) {
            btn.classList.remove('active');
            nav.classList.remove('active');
            document.body.classList.remove('menu-open');
        }
    });
})();

7. Висновки

Під час виконання лабораторної роботи №3 було реалізовано верстку власного сайту з повною відповідністю всім вимогам щодо адаптивності та функціональності.

Основні досягнення:

  1. Створено семантичну структуру HTML з використанням тегів <header>, <nav>, <section>, <footer>
  2. Реалізовано адаптивну навігацію з бургер-меню, яка трансформується на пристроях з малою роздільною здатністю
  3. Застосовано CSS Grid та Flexbox для створення адаптивної сітки контенту
  4. Додано медіа-запити для адаптації стилів під різні розміри екранів (десктоп, планшет, смартфон)
  5. Використано відносні одиниці вимірювання (% , rem, em) для забезпечення гнучкості інтерфейсу
  6. Реалізовано візуальні ефекти та анімації (плавне відкриття меню, зміна стану елементів при наведенні)
  7. Організовано логічну файлову структуру з окремими файлами для HTML, CSS, JavaScript та ресурсів

Отримані знання та навички дозволяють створювати сучасні адаптивні веб-сайти, які коректно відображаються на різних пристроях та розмірах екранів. Розуміння принципів адаптивної верстки, використання медіа-запитів та реалізація інтерактивних елементів є важливими для створення доступних та зручних веб-додатків.
