Тема, Мета, Місце розташування сайту, звіту

Тема: «Веб-орієнтований застосунок магазину книг «Bookerino».»

Мета: Зробити зручний і простий у використанні інтернет-магазин книг «Bookerino», який дозволяє користувачам легко переглядати каталог книг та аксесуарів, швидко оформлювати замовлення та отримувати доставку. Сайт покликаний забезпечити приємний інтерфейс, показати асортимент із фото та описами, а також спростити процес вибору й покупки книг.

Посилання на виконані завдання

  • Репозиторій власного веб-застосунку (GitHub): посилання
  • Власний веб-застосунок (Жива сторінка): посилання
  • Репозиторій звітного HTML-документа (GitHub): посилання
  • Звітний HTML-документ (Жива сторінка): посилання
  • Репозиторій самостійної роботи №1 (GitHub): посилання
  • Самостійна робота №1 (Жива сторінка): посилання

Опис предметного середовища. Опис бізнес-логіки

Предметна галузь: інтернет-магазин книг «Bookerino». Сайт надає каталог книг та аксесуарів, інформацію про компанію, контактні дані та можливість оформити замовлення онлайн.

Структура веб-застосунку

  • Головна сторінка — вітальний блок із спеціальними пропозиціями та коротка інформація.
  • Каталог — перелік книг з назвою, зображенням, ціною й описом.
  • Про нас — відомості про компанію та послуги.
  • Відгуки — перегляд і додавання відгуків.
  • Контакти — адреса, графік роботи, телефони/пошта.
  • Доставка та оплата — інформація про способи доставки та оплати.

Сценарій взаємодії (бізнес-логіка)

  1. Користувач відкриває сайт і переглядає каталог книг та аксесуарів.
  2. Ознайомлюється з книгами, обирає потрібні товари.
  3. Заповнює форму замовлення (ПІБ, телефон, дата/адреса доставки, примітки).
  4. Магазин отримує замовлення та підтверджує його (наприклад, телефоном).
  5. Кур'єр доставляє замовлення клієнту у вказаний час.
  6. Після отримання користувач може залишити відгук на сайті.

Функціональні вимоги

  • Відображення каталогу товарів з цінами, фото та описами.
  • Перегляд детальної інформації про книгу.
  • Форма оформлення замовлення.
  • Форма додавання відгуку.
  • Розділи «Про нас» і «Контакти» доступні з головної навігації.
  • Адаптивна верстка для ПК і мобільних пристроїв.

Нефункціональні вимоги

  • Простий, інтуїтивний інтерфейс.
  • Час завантаження сторінки — орієнтовно до 3 секунд.
  • Коректна передача даних із форм.
  • Підтримка сучасних браузерів (Chrome, Firefox, Edge, Safari).
  • Використання якісних зображень товарів.

Стек технологій

  • HTML5 — семантична розмітка сторінок і контент-блоків.
  • CSS3 — стилізація, сітка/флекс-верстка, адаптивність (media queries).
  • JavaScript (Vanilla) — навігація, генерація оглавлення, плавний скрол, взаємодія з формами.
  • Git + GitHub — контроль версій, командна робота, issues/PR.
  • GitHub Pages — публікація статичного звіту/сайту.
  • Assets — зображення товарів та іконки у assets/images/.
  • Форматування коду — узгоджені правила (Prettier/ESLint за потреби).

Висновки

У ході оформлення звіту для «Bookerino» я вибудував чітку структуру сторінок, застосував семантичну HTML-розмітку та налаштував автогенерацію оглавлення з коректним позиціонуванням під час скролу. Додані приклади таблиць, зображень і форм відтворюють реальний вміст проєкту, а стилі для код-блоків із перенесенням рядків забезпечують читабельність без горизонтального прокрутання. Також вирішено проблему відносних шляхів у завантажуваних фрагментах за допомогою постобробки посилань у JavaScript, що гарантує стабільне відображення ресурсів. Загалом отримано цілісний, зрозумілий та придатний до публікації шаблон звіту.
